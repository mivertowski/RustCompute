## Per-spec results

### `hlc.tla` — Hybrid Logical Clocks

- Constants: `Nodes = {n1, n2}`, `MaxPhysical = 3`, `MaxEvents = 4`.
- Invariants: `TypeOK`, `PhysicalTimeMonotonic`, `CausalOrdering`,
  `PerNodeMonotonic`, `EventCountsBounded`.
- Outcome: safety invariants hold; physical-time component never
  regresses, per-node emission order is strictly increasing, and
  every delivered timestamp is < the receiver's post-receive clock.

### `k2k_delivery.tla` — Single-GPU K2K ordering

- Constants: `Actors = {a1, a2}`, `MaxMsgs = 4`, `QueueCap = 2`.
- Invariants: `TypeOK`, `QueueBoundOK`, `DeliveredImpliesAccepted`,
  `NoMessageLoss`, `FIFOPerSender`, `NoDuplicateDelivery`,
  `DropAcceptedDisjoint`.
- Outcome: FIFO-per-sender preserved, no duplicates, queue cap
  respected.

### `migration.tla` — 3-phase actor migration

- Constants: `MaxMsgs = 4`.
- Invariants: `TypeOK`, `PhaseLegal`, `AtomicSwap`, `NoMessageLoss`,
  `FIFOPreserved`, `ChecksumMatch`.
- Notes for v1.1: the spec previously had a subtle model-level bug —
  `AcceptDuringQuiesce` kept mutating `src_state` after
  `CaptureState` snapshotted the checksum, so the source and target
  checksums diverged when late messages arrived during transfer.
  The spec was patched to introduce an explicit `captured_state`
  variable (the immutable snapshot taken at `CaptureState`), and
  `Transfer` now copies from `captured_state`. This mirrors the real
  Rust implementation (`crates/ringkernel-cuda/src/multi_gpu/runtime.rs`),
  which captures the staging buffer as a whole and both sides
  compute CRC32 over the same bytes. After the fix, `ChecksumMatch`
  holds under the full `MaxMsgs = 4` bound.

### `multi_gpu_k2k.tla` — Cross-GPU K2K routing

- Constants: `GPUs = {g1, g2}`, `Actors = {a1, a2, a3}`,
  `ActorGpu <- DefaultActorGpu`, `NVLinkedGPUs = {g1, g2}`,
  `MaxMsgs = 3`.
- Invariants: `TypeOK`, `DeliverySemantics`, `BoundedLatency`.
- Outcome: every accepted message is locatable somewhere in the
  pipeline, and no NVLink-adjacent pair routes via host staging
  (by construction).

### `tenant_isolation.tla` — Per-tenant K2K partitioning

- Constants: `Tenants = {t1, t2}`, `Kernels = {k1, k2, k3}`,
  `MaxMsgs = 3`.
- Invariants: `TypeOK`, `NoSameMessageInBothTenants`,
  `CrossTenantRejected`, `IsolationByConstruction`,
  `InboxesWellFormed`.
- Outcome: cross-tenant sends are always rejected; no message is
  observable in two tenants' inboxes at once.

### `actor_lifecycle.tla` — Actor state machine

- Constants: `Actors = {a1, a2, a3}`, `Parent <- DefaultParent`,
  `MaxSteps = 8`.
- Invariants: `TypeOK`, `StateTransitionsLegal`,
  `ChildActorsTerminate`, `RestartPreservesState`,
  `SnapshotMonotonic`.
- Outcome: supervisory termination (parent's destroy requires all
  children terminated first), restart preserves snapshot, snapshots
  never exceed live state.

## Configuration fixes applied for v1.1

Several `.cfg` files had TLC-parser issues that prevented them from
loading on TLC 2.19. These were fixed without relaxing any invariant:

1. **`actor_lifecycle.cfg`** — `Parent = (a1 :> "ROOT" @@ ...)` is
   not valid cfg syntax; replaced with `Parent <- DefaultParent`
   where `DefaultParent` is a computed operator inside the .tla.
2. **`multi_gpu_k2k.cfg`** — same `:>` / `@@` issue; replaced with
   `ActorGpu <- DefaultActorGpu`.
3. **`multi_gpu_k2k.tla`** — `\E a \in S, i \in T : …` comma form
   replaced with nested `\E` for TLC 2.19 strict-mode parser.
4. **All six `.cfg` files** — added `CHECK_DEADLOCK FALSE` because
   every bounded model reaches a terminal state (no action enabled)
   once its `MaxMsgs`/`MaxEvents`/`MaxSteps` bound saturates. TLC's
   default deadlock detector otherwise reports those terminal
   states as bugs, even though they are the expected completion of
   the bounded run.

The invariants themselves were not altered.

## Reproduction

```bash
# Download tla2tools.jar (needs Java 17+)
curl -fsSL -o docs/verification/tla2tools.jar \
  https://github.com/tlaplus/tlaplus/releases/latest/download/tla2tools.jar

# Run the paper's Experiment 5 wrapper (produces a CSV summary):
TLC_CMD="java -XX:+UseParallelGC -jar docs/verification/tla2tools.jar" \
  TLC_WORKERS=8 \
  docs/paper/experiments/05-tlc-stats/run.sh /tmp/tlc_out
# Per-spec logs under /tmp/tlc_out/tlc_<spec>.log
# Summary CSV at /tmp/tlc_out/tlc_stats.csv
```

Or run a single spec:

```bash
cd docs/verification
java -XX:+UseParallelGC -jar tla2tools.jar -workers 8 \
  -config migration.cfg migration.tla
```
