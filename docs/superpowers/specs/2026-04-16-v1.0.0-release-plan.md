# RingKernel v1.0.0 Release Plan

**Date:** 2026-04-16
**Goal:** First production-grade release. NVIDIA CUDA focus. Polished, honest, working.

---

## Principles

1. **Ship what works.** Remove scaffolds, stubs, and backends that don't deliver real GPU execution.
2. **NVIDIA-first.** CUDA persistent actors are proven (8,698x vs traditional). Everything else is a distraction for v1.0.
3. **Honest docs.** Every claim backed by benchmarks. No "100% complete" when it's 75%.
4. **Working examples.** Every example compiles, runs, and demonstrates a real use case.
5. **Clean repo.** No dead code, no orphan features, no misleading feature flags.

---

## v1.0.0 Changelog Draft

### Added
- Hopper H100 support: Thread Block Clusters, DSMEM, TMA, Green Contexts
- GpuArchitecture::blackwell() preset for B200 (sm_100)
- Architecture auto-detection (RINGKERNEL_CUDA_ARCH env var)
- cudarc 0.19.3 with CUDA 13.x compatibility
- libcu++ ordered atomics (default for persistent kernels)
- CudaRuntime::launch() bridges to PersistentSimulation for real GPU execution
- Graceful shutdown handler (SIGTERM/SIGINT)
- CheckpointManager for periodic actor state snapshots
- Per-actor metrics through runtime API (all backends)
- Dynamic actor scheduling framework (scheduler warp + work stealing codegen)
- CloudWatch audit sink (feature-gated)
- OTLP export via dedicated `otel` feature flag
- Academic benchmark framework with statistical rigor
- 8 compile-fail tests for proc macros

### Changed
- All typed error enums (AccNetError, WaveSimError, TxMonError, ProcIntError, WaveSim3dError)
- Zero bare `.unwrap()` in production code
- `clippy::unwrap_used` warning on 12 crates
- `cargo-audit` in CI pipeline
- println/eprintln migrated to structured tracing
- 24 unsafe blocks documented with SAFETY comments

### Removed
- WebGPU backend (`ringkernel-wgpu`, `ringkernel-wgpu-codegen`)
- Metal backend (`ringkernel-metal`)
- WaveSim3D (`ringkernel-wavesim3d`) — hard wgpu dependency
- All wgpu/metal feature flags from remaining crates

### Fixed
- TLS PEM certificate parsing (was placeholder returning empty vectors)
- XOR crypto fallback now emits deprecation warning
- CLI project name validation (removed unsafe unwrap)
